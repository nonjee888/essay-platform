# 수필 연재 플랫폼 — 설계 (v1)

작성일: 2026-07-22

## 목적

내가 준 **줄거리**를 AI가 **수필체로 각색**해 정적 웹사이트에 연재한다.
저자·사이트는 **익명**. 사이트 제목은 **"묵혀둔 이야기"**, 부제 "오래 삭여 두었던 글을 하나씩 꺼냅니다".

## 스택

- **Astro** — 마크다운 콘텐츠 컬렉션, 이미지 자동 최적화, RSS, 목록 생성이 모두 기본 기능.
- **배포:** Vercel 또는 Netlify (git push → 자동 배포).
- **댓글:** Giscus (GitHub Discussions 기반, 무료, 백엔드 없음).
- 서버·DB·로그인 없음 (정적).

## 디자인

**A · 문학적 미니멀**: 흰 배경, 세리프(명조) 글꼴, 넉넉한 여백, 포인트 색은 브라운(#b08d57) 계열.
글 자체에 집중하는 책/문예지 느낌.

## 전체 흐름

```
줄거리 입력 → [로컬 생성 스크립트] → 초안 .md (draft:true) → 내가 읽고 수정
→ draft 해제 → git push → 자동 배포
```

AI는 **자동 발행하지 않는다.** 항상 `draft: true`로 생성하고, 사람이 확인 후 직접 공개한다.

## 구성 요소

### 1. 콘텐츠 컬렉션 `src/content/essays/*.md`

글 1편 = 마크다운 파일 1개. frontmatter 스키마:

| 필드 | 필수 | 설명 |
|---|---|---|
| `title` | ✓ | 글 제목 |
| `date` | ✓ | 발행일 |
| `series` | ✓ | 연재물 이름 (예: "서른의 계절") |
| `episode` | ✓ | 회차 번호 (정수) |
| `tags` | ✓ | 태그 배열 |
| `draft` | ✓ | true면 미공개 |
| `cover` | · | 대표 사진 경로 (선택) |

- 스키마는 Astro `content config`(zod)로 검증 — 필드 누락·타입 오류 시 빌드 실패.
- 연재 순서는 `series` + `episode`로 정한다.
- 사진은 `src/assets/` 또는 글 옆 폴더에 두고 마크다운에서 참조 → 빌드 시 자동 최적화. 업로드 서버 없음.

### 2. AI 생성 스크립트 `scripts/generate.mjs` (로컬 전용)

- 실행: `node scripts/generate.mjs` — 줄거리를 프롬프트로 묻거나 `outline.txt`에서 읽음.
- Anthropic SDK로 Claude에 "이 줄거리를 수필체로 각색" 요청 → 위 frontmatter 형식의 `.md` 초안 파일 생성 (`draft: true`).
- 모델: 최신 Claude (`claude-opus-4-8` 기본).
- **API 키는 사용자 환경변수 `ANTHROPIC_API_KEY`에서만 읽는다.** 코드·커밋 파일에 키를 넣지 않는다. `.env`는 `.gitignore`.
- 각색은 원 줄거리의 사실을 왜곡하지 않는다. 생성 후 반드시 사람이 검토한다.

### 3. 페이지

| 경로 | 내용 |
|---|---|
| `/` | 최신 글 + 연재물 목록 |
| `/essays/[slug]` | 글 본문 + 하단 Giscus 댓글 + 이전/다음 회차 이동 |
| `/series/[name]` | 한 연재물의 회차 목록 (1화, 2화…) |
| `/tags/[tag]` | 태그별 목록 |
| `/rss.xml` | 자동 생성 피드 |

`draft: true` 글은 목록·빌드에서 제외.

## v1에서 뺀 것 (→ v2)

- **메일 알림**: 외부 뉴스레터 서비스(Buttondown 등) 필요. RSS로 당분간 대체.
- **카카오 알림**: 알림톡은 사업자등록 + 비즈니스 채널 + 유료 대행사 필요. 비추천, 조건 갖춰지면 재검토. 당분간 RSS를 카카오 오픈채팅에 수동 공유.
- 관리자 로그인/DB: 정적 사이트라 불필요.

## 보안·안전 정책 (프로젝트 CLAUDE.md에도 반영)

- API 키·비밀번호·토큰을 코드/커밋 파일에 넣지 않는다. 키는 `ANTHROPIC_API_KEY` 환경변수에서만.
- AI 생성 글은 자동 발행하지 않는다 (`draft:true` → 사람이 공개).
- 실제 발행·배포(git push, 도메인 연결)는 사용자 허락 후 진행.
- 고객명·계약·개인정보 등 민감 데이터는 글·저장소에 넣지 않는다.

## 검증

- Astro content config(zod) 스키마가 frontmatter를 빌드 시 검증.
- `scripts/generate.mjs`에 생성 결과의 frontmatter가 스키마에 맞는지 확인하는 자체 체크 1개 포함 — 형식이 깨지면 실패.
